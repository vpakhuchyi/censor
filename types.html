Types Support

This guide details which Go types are supported by Censor and how they are handled.

Supported Types​

Basic Types​

TypeSupportNotes
string✅Full support with pattern matching
bool✅Values displayed as-is
int/int8/int16/int32/int64✅Values displayed as-is
uint/uint8/uint16/uint32/uint64✅Values displayed as-is
float32/float64✅Values displayed as-is
complex64/complex128✅Values displayed as-is
byte (uint8)✅Values displayed as-is
rune (int32)✅Values displayed as-is

Composite Types​

TypeSupportNotes
struct✅Full support with field tags
map✅Both keys and values processed
slice✅All elements processed
array✅All elements processed
pointer✅Underlying value processed
interface✅Concrete type processed
channel❌Not supported
func❌Not supported

Special Types​

TypeSupportNotes
time.Time✅Processed as string
encoding.TextMarshaler✅Custom text representation
error✅Error message processed
nil✅Displayed as "nil"

Detailed Behavior​

Structs​

Structs are processed field by field:

type User struct {
    ID       string `censor:"display"` // Will be visible
    Password string                    // Will be masked
    private  string                    // Ignored (unexported)
}

Notes:

  • Use censor:"display" tag to show field value
  • Unexported fields are ignored
  • Embedded structs are processed recursively

Maps​

Maps are processed recursively for both keys and values:

// Simple map
map[string]string{"password": "secret"} 
// Output: map[password: [CENSORED]]

// Complex map
map[User][]string{
    {ID: "123"}: {"data1", "data2"},
}
// Output: map[{ID: 123, ...}: [[CENSORED], [CENSORED]]]

Notes:

  • Map type can be displayed with configuration
  • Keys and values processed independently
  • Nested maps handled recursively

Slices and Arrays​

Elements are processed according to their type:

// String slice
[]string{"password1", "password2"}
// Output: [[CENSORED], [CENSORED]]

// Mixed slice
[]interface{}{1, "secret", true}
// Output: [1, [CENSORED], true]

Pointers​

Pointers are dereferenced and their values processed:

s := "secret"
ptr := &s
// Output: [CENSORED]

var nilPtr *string
// Output: nil

Interfaces​

Interface values are processed based on their concrete type:

var data interface{} = "sensitive"
// Output: [CENSORED]

var data interface{} = 42
// Output: 42

Unsupported Types​

The following types are not supported by Censor:

  1. Channels

    ch := make(chan int)
    // Will not be processed

  2. Functions

    fn := func() {}
    // Will not be processed

  3. Unsafe Pointer

    var ptr unsafe.Pointer
    // Will not be processed

Best Practices​

  1. Type Safety

    • Use appropriate types for your data
    • Avoid interface unless necessary
    • Use struct tags consistently

  2. Performance

    • Consider type complexity when processing large data structures
    • Use pointers for large structs
    • Avoid unnecessary type conversions

  3. Error Handling

    • Check for unsupported types before processing
    • Handle nil values appropriately
    • Use appropriate error logging

  4. Custom Types

    • Implement encoding.TextMarshaler for custom string representation
    • Use struct tags for custom type fields
    • Document type-specific behavior

Common Pitfalls​

  1. Unexported Fields

    • Unexported struct fields are always ignored
    • Use exported fields for data that needs processing

  2. Interface Type Assertions

    • Be careful with type assertions in interface values
    • Handle unknown types appropriately

  3. Circular References

    • Avoid circular references in data structures
    • Use pointers carefully to prevent infinite recursion

  4. Complex Types

    • Be aware of performance impact with deeply nested structures
    • Consider flattening complex data structures when possible
