Logger Integration

Censor provides built-in handlers for popular logging frameworks, making it easy to protect sensitive data in your logs.

Zap Logger Integration​

The github.com/vpakhuchyi/censor/handlers/zap package provides a configurable handler for the go.uber.org/zap library. It allows you to apply censoring to log entries before they reach the core logger.

Basic Setup​

package main

import (
    censorlog "github.com/vpakhuchyi/censor/handlers/zap"
    "go.uber.org/zap"
    "go.uber.org/zap/zapcore"
)

type User struct {
    Name  string `censor:"display"`
    Email string
}

func main() {
    // Create a new Censor handler with the default configuration
    o := zap.WrapCore(func(core zapcore.Core) zapcore.Core {
        return censorlog.NewHandler(core)
    })

    // Create a zap logger with the wrapped core
    l, err := zap.NewProduction(o)
    if err != nil {
        panic(err)
    }

    user := User{Name: "John Doe", Email: "john@example.com"}

    // Log as usual - sensitive data will be automatically masked
    l.Info("user details", zap.Any("user", user))
    // Output: {"level":"info",...,"msg":"user details","user":"{Name: John Doe, Email: [CENSORED]}"}
}

Configuration Options​

The Censor handler for Zap provides several configuration options:

  1. Custom Censor Instance
handler := censorlog.NewHandler(
    core,
    censorlog.WithCensor(customCensor),
)
  1. Message Formatting
handler := censorlog.NewHandler(
    core,
    censorlog.WithMessagesFormat(), // Enable censoring of log messages
)
  1. Key Formatting
handler := censorlog.NewHandler(
    core,
    censorlog.WithKeysFormat(), // Enable censoring of log keys
)

Data Processing​

The handler processes different parts of log entries:

  • msg: The log message itself (optional, requires WithMessagesFormat)
  • key: Key names in structured logging (optional, requires WithKeysFormat)
  • value: The actual values being logged (always processed)

Supported Logger Patterns​

Non-Sugared Logger​

logger.Info(msg string, fields ...zap.Field)
logger.With(fields ...zap.Field).Info(msg string)

Sugared Logger​

logger.Info(args ...interface{})
logger.Infof(template string, args ...interface{})
logger.Infow(msg string, keysAndValues ...interface{})
logger.With(args ...interface{}).Info(args ...interface{})

Slog Logger Integration​

Censor also provides integration with Go's standard log/slog package. Here's how to use it:

package main

import (
    "log/slog"
    "os"
    
    "github.com/vpakhuchyi/censor"
    censorlog "github.com/vpakhuchyi/censor/handlers/slog"
)

func main() {
    // Create a new slog handler with censoring
    handler := censorlog.NewHandler(
        slog.NewTextHandler(os.Stdout, nil),
    )

    // Create a logger with the censoring handler
    logger := slog.New(handler)

    // Your sensitive data
    data := map[string]string{
        "username": "john_doe",
        "password": "secret123",
    }

    // Log with automatic censoring
    logger.Info("user data", "details", data)
    // Output will mask sensitive fields
}

Best Practices​

  1. Initialize Early: Set up the Censor handler during logger initialization
  2. Configure Appropriately: Use the configuration options that match your security requirements
  3. Test Thoroughly: Verify that sensitive data is properly masked in your logs
  4. Monitor Performance: While Censor is optimized for performance, monitor its impact in your specific use case
  5. Keep Updated: Stay current with the latest version for security updates and new features
